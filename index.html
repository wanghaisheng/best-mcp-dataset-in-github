About The Tool

The MCP GitHub Popular Repo Discovery Tool is a command-line utility designed to help you efficiently find, categorize, and analyze popular GitHub repositories. While it has a special focus on Minecraft (MCP) projects, it's versatile enough for any keyword-based GitHub search.

This tool automates the search and data collection process, providing structured information including repository names, descriptions, star/fork counts, extracted keywords, and preliminary categorization and tech stack identification. Our vision is to simplify the discovery of valuable open-source projects for developers, researchers, and enthusiasts.

Accessing Datasets & Collections

This tool generates detailed datasets in JSON format, allowing for easy analysis and integration with other applications. Each dataset is typically saved with a date suffix (e.g., results/data_YYYY-MM-DD.json) in the results directory after running the script.

How to Generate Datasets:

  1. Configure Environment: Ensure you have Python installed and the necessary libraries (requests, python-dotenv). Set up your .env file with your GITHUB_TOKEN, target KEYWORDS_ENV (e.g., "mcp server"), MIN_STARS, MIN_FORKS, and OUTPUT_DIR.
  2. Run the Script: Execute python dailysaasstarter.py from your terminal.
  3. Find Your Data: Check the specified output directory (default: results/) for your freshly generated JSON file.

Example: After running the script for "mcp server" keywords, you might find a file like results/data_2023-10-28.json containing a collection of discovered repositories.

(Note: This website provides information about the tool. The actual datasets are generated locally on your machine by running the Python script.)

Core Features

Keyword-Based GitHub Search

Performs searches on GitHub for repositories matching user-defined keywords.

Data Extraction & Filtering

Extracts key information (name, description, URL, stars, forks) and allows filtering by minimum stars and forks.

Automated Categorization

Assigns categories to repositories based on keywords found in their descriptions (e.g., "server_core_bukkit_like", "proxy" for MCP).

Tech Stack Identification

Identifies potential technologies used in the repositories based on keywords (e.g., "java", "spigot", "forge").

Dated JSON Output

Saves the collected and processed data into a JSON file with a date suffix for historical tracking and easy integration.
